HTTP Methods with Examples

Method Description Example
GET Retrieve data from the server. Safe and idempotent. GET /users/johndoe HTTP/1.1
POST Submit data to be processed. May cause a change in server state. POST /users HTTP/1.1
Content-Type: application/json
{ "username": "newuser", "password": "password123" }
PUT Update a resource on the server. Replaces or creates a new one. PUT /users/johndoe HTTP/1.1
Content-Type: application/json
{ "name": "John Doe", "email": "john.doe@example.com" }
PATCH Apply partial modifications to a resource. PATCH /users/johndoe HTTP/1.1
Content-Type: application/json
{ "email": "newemail@example.com" }
DELETE Delete a specified resource on the server. DELETE /users/johndoe HTTP/1.1
HEAD Retrieve only the headers of a resource. HEAD /users/johndoe HTTP/1.1
OPTIONS Retrieve information about the communication options for the target resource. OPTIONS /users/johndoe HTTP/1.1
TRACE Echo the received request to test and troubleshoot. TRACE /users/johndoe HTTP/1.1
CONNECT Establish a transparent TCP/IP tunnel for SSL-encrypted communication. CONNECT server.example.com:443 HTTP/1.1
PROPFIND Retrieve properties (metadata) of a resource. PROPFIND /files/docs HTTP/1.1
PROPPATCH Apply partial modifications to the properties of a resource. PROPPATCH /files/docs HTTP/1.1
Content-Type: application/xml
<propertyupdate xmlns="DAV:">...</propertyupdate>
MKCOL Create a new collection (e.g., a new directory). MKCOL /files/images HTTP/1.1
COPY Copy a resource from one URI to another. COPY /files/docs/report.txt HTTP/1.1
Destination: /archive/report.txt
MOVE Move a resource from one URI to another. MOVE /files/docs/report.txt HTTP/1.1
Destination: /archive/report.txt
LOCK Establish a lock on a resource. LOCK /files/docs/report.txt HTTP/1.1
UNLOCK Release a lock on a resource. UNLOCK /files/docs/report.txt HTTP/1.1
REPORT Request information about the resources contained in a collection. REPORT /files/docs HTTP/1.1
MKACTIVITY Create a new activity. MKACTIVITY /version-control/activity123 HTTP/1.1
MERGE Perform a logical merge of changes into a new resource. MERGE /version-control/activity123 HTTP/1.1
CHECKOUT Check out a resource for exclusive editing. CHECKOUT /version-control/docs/report.txt HTTP/1.1
CHECKIN Check in a resource after exclusive editing. CHECKIN /version-control/docs/report.txt HTTP/1.1
LABEL Add a label to a resource. LABEL /version-control/docs/report.txt HTTP/1.1
UPDATE Update a working resource to reflect the current state. UPDATE /version-control/docs/report.txt HTTP/1.1
VERSION-CONTROL Version control a resource. VERSION-CONTROL /version-control/docs/report.txt HTTP/1.1
BASELINE-CONTROL Specify the DAV compliance of the baseline for a version-controlled resource. BASELINE-CONTROL /version-control/docs/report.txt HTTP/1.1
